# Contributing to Single Cell Polygenic Risk Score

Thank you for your interest in contributing! Here are some guidelines to help you get started.

## Code Style

- Follow PEP 8 guidelines for Python code
- Use meaningful variable and function names
- Add docstrings to all functions and classes
- Keep lines to a reasonable length (max 100 characters)

## Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request

## Reporting Bugs

Use the GitHub Issues tab and select the Bug Report template. Include:
- A clear description of the bug
- Steps to reproduce it
- Expected vs actual behavior
- Environment details (OS, Python version, etc.)

## Suggesting Enhancements

Feel free to suggest improvements by opening an issue with details about:
- What you'd like to see
- Why it would be useful
- Any implementation ideas

## Testing

Before submitting a PR, ensure:
- Your code runs without errors
- Any new functions are tested
- Output files are correctly generated

## Questions?

Feel free to open an issue if you have questions about how to contribute.
